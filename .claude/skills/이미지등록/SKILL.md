# 이미지등록 — 사진 ↔ 상품 자동 매칭 → RPA 등록

토스포스 매장의 "사진 넣어주세요" 요청을 처리한다. **조회(RPA) → 매칭(비전) → 등록(RPA)** 순서로, 신뢰도 높은 매칭만 자동 등록하고 애매한 건 검수로 뺀다. (메뉴 텍스트 등록은 [[메뉴판독]], 요청 적재는 대시보드 메뉴등록 카테고리 참고)

## 전제
- `rpa/` 셋업 완료 + 로그인 세션(`rpa/.auth/state.json`) 유효. 만료 시 `node rpa/login.js`.
- 이미지 파일 확보: 슬랙 첨부는 `slack_read_file`(비전 판독 겸용), 로컬 저장이 필요하면 담당자에게 다운로드 요청. 고객 Drive 링크는 접근 불가할 수 있음 → 슬랙 첨부 우선.

## 실행 절차

1. **상품 목록 조회** —
   ```
   node rpa/list-products.js --biz <사업자번호> --store "<상호명>" --out /tmp/products.json
   ```
   등록 상품 전체(이름·가격)가 JSON으로 나온다 (페이지네이션 자동 순회).

2. **사진 판독 & 매칭** — 각 이미지를 비전으로 보고 아래를 수행:
   - 사진 속 음식/상품이 무엇인지 판독 (파일명·캡션·요청문 힌트 병용)
   - 1번의 상품명 목록과 매칭 → `{ image, product, confidence(high/low), note }`
   - **추정 금지 원칙**: 확신 없으면 `low` + 후보 2~3개 표기. 동일 상품에 사진 여러 장이면 가장 선명한 1장.

3. **매칭 결과 보고 & 검수** — 표(이미지 | 매칭 상품 | 신뢰도 | 비고)로 보고.
   - `high` → 자동 등록 대상 / `low` → 담당자 선택 대기
   - 이미 이미지가 있는 상품은 현재 교체 미지원 → 목록에서 제외하고 표기

4. **등록 (상품별 반복)** — 먼저 dry-run으로 1건 검증 후, 승인 받으면 `--confirm`:
   ```
   node rpa/upload-menu.js --biz <사업자번호> --store "<상호명>" --product "<상품명>" --images <이미지> [--confirm]
   ```
   - 각 실행의 스크린샷(`rpa/shots/`)을 근거로 결과 보고
   - 실패(플레이스홀더 없음=이미지 이미 존재 등)는 사유와 함께 스킵 목록으로

5. **마무리** — 등록 N건 / 검수대기 N건 / 스킵 N건 요약. 슬랙 요청 건이면 처리 완료 리액션은 담당자 확인 후.

## 주의
- **테스트 먼저**: 새 매장 유형이면 첫 1건은 반드시 dry-run 스크린샷 검수 후 --confirm.
- 음식 사진만으로 메뉴명 추정은 한계가 있으므로 low 비율이 높으면 무리하지 말고 담당자에게 "사진별 메뉴명 캡션"을 요청하는 게 빠르다.
- 대량(10장+)일 때는 5장씩 끊어 실행하고 매 배치 스크린샷 확인 (봇 탐지·오류 파급 방지).
