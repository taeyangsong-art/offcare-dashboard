# 원격 AS요청 채널 → 대시보드 자동 집계

슬랙 `#0_원격_as_요청` 채널(ID: `C09HRUSG4TX`)의 **오늘** 메시지를 읽어, 리액션 이모지로 직원별 처리건수를 집계하고 `index.html`의 `SLACK_IMPORT` 데이터를 갱신한다.

## 이모지 규칙
- **담당자**: `원격규빈`=김규빈, `원격선유`=배선유, `원격성현`=심성현, `원격동욱`=김동욱, `원격현기`=김현기, `원격태양`=송태양, `원격기범`=김기범, `원격상원`=서상원, `원격민석`=최민석
- **카테고리**: `원격as`=AS, `원격온보딩`=온보딩 (있으면 `원격명의변경`=명의변경, `원격메뉴등록`=메뉴등록, `원격voc`=VOC, `원격배달`=배달)
- **집계 규칙**: 한 메시지 = 1건. `담당자 이모지 + 카테고리 이모지`가 있으면 해당 직원·카테고리 +1 (담당자만 있고 카테고리 이모지 없으면 채널 기본 = AS).
- **확인필요(pending)**: 담당자 이모지 없음 / 카테고리 이모지 없음 / `1차_부재`·`2차_부재` 이모지가 찍힌 건. (부재는 담당자가 있어도 완료로 세지 않고 pending 으로 분류)
- `원격솔라피발송`, `_확인` 류는 진행상태이므로 집계에서 제외. `원격외주`는 대시보드 카테고리에 없어 참고로만 표기.

## 실행 절차

1. **오늘 날짜 확인** — 시스템 오늘 날짜를 `YYYY-MM-DD`로 확정한다 (예: 2026-07-08).

2. **채널 메시지 읽기** — `slack_read_channel`(channel_id `C09HRUSG4TX`, `response_format: detailed`, `limit: 100`)로 최신 메시지를 읽는다.
   - 결과가 커서 파일로 저장되면(대개 그렇다) 그 **파일 경로**를 확보한다.
   - 반환된 가장 오래된 메시지가 아직 **오늘**이면, `cursor`로 다음 페이지를 더 읽어 어제로 넘어갈 때까지 반복한다. (여러 파일이면 순서대로 처리)

3. **집계 + 데이터 파일 갱신** — 저장된 파일 경로와 오늘 날짜로 실행 (`--write` 로 `slack-data.js` 자동 갱신):
   ```
   node scripts/tally-remote-as.js "<슬랙읽기결과파일경로>" <YYYY-MM-DD> --write slack-data.js
   ```
   - 스크립트가 `slack-data.js` 의 `days["<오늘>"]` 항목을 **추가/갱신**하고 `version` 을 자동으로 +1 한다. (index.html 은 수정하지 않는다 — 데이터는 전부 `slack-data.js` 에만 있다)
   - **여러 파일**(페이지네이션)인 경우: 가장 오래된 파일부터 순서대로 `--write` 없이 실행해 각 결과를 확인한 뒤, 하나로 합쳐 한 번만 `--write` 하거나, 파일들을 이어붙인 임시 파일에 대해 1회 실행한다. (같은 날짜의 counts 는 합산, pending 은 상호+사업자 기준 중복 제거)

4. **검증** — 데이터 파일과 대시보드 문법 확인:
   ```
   node -e "global.window={};new Function('window',require('fs').readFileSync('slack-data.js','utf8'))(global.window);console.log('days:',Object.keys(global.window.SLACK_DATA.days),'v',global.window.SLACK_DATA.version)"
   ```

5. **요약 보고** — 완료 건수, 카테고리별·직원별 요약, 확인필요 건수(상호 목록), 그리고 `slack-data.js` 누적 날짜 목록을 한국어 표로 정리해 보고한다.

## 주의
- 이모지 매핑에 없는 새 담당자/카테고리 이모지가 보이면, 집계에서 누락됨을 보고하고 `scripts/tally-remote-as.js`의 `personMap`/`catMap`에 추가를 제안한다.
- 데이터는 **`slack-data.js` 에만** 누적된다. 대시보드(`index.html`)는 로드 시 이 파일을 읽어 각 날짜의 AS/온보딩 수치를 반영한다. (해당 날짜의 AS/온보딩은 슬랙이 원본이라 매 로드 시 덮어씀. 다른 카테고리·수동 입력값은 건드리지 않음)
- 같은 날 여러 번 실행하면 그 시점까지의 최신 실적으로 그 날짜 항목을 갱신한다.
